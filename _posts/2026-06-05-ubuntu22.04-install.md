# HyperV创建Ubuntu虚拟机

## HyperV创建虚拟机步骤：

1、打开Hyper-V设置，修改虚拟硬盘和虚拟机默认文件夹（不修改默认C盘）
![hyperv-setting](/styles/images/Linux/ubuntu/hyperv-setting.png)

2、编辑磁盘.选择操作里面扩展，修改磁盘大小为128G（默认12G）
![edit-dev](/styles/images/Linux/ubuntu/edit-dev.png)

3、快速新建
![quickly-start](/styles/images/Linux/ubuntu/quickly-start.png)


## 硬盘重新分区

### 1. 查看当前分区表

首先确认磁盘信息，避免误操作：`sudo fdisk -l /dev/sda`这会显示当前分区情况，确保操作对象正确
![sudo-fdisk-l](/styles/images/Linux/ubuntu/sudo-fdisk-l.png)

### 2. 进入交互式分区模式

`sudo fdisk /dev/sda` 输入后进入fdisk交互界面，显示Command (m for help)提示符
![sudo-fdisk](/styles/images/Linux/ubuntu/sudo-fdisk.png)

### 3. 删除旧分区（可选）

若需重新分区，先删除现有分区：输入d删除分区，按提示选择分区号

重复操作直到所有分区删除。
### 4. 创建新分区

常用分区方案示例（按需调整）：

**方案A**：单根分区（推荐简单场景）

输入n创建新分区，选择p（主分区），分区号默认1，起始扇区直接回车（默认2048）

结束扇区输入+128G分配全部空间

**方案B**：多分区（例如根目录+home）

### 5. 保存并退出

输入p预览分区表

确认无误后输入w写入并退出

### 6. 后续操作

分区后需格式化和挂载（以ext4为例）：

格式化：`sudo mkfs.ext4 /dev/sda1`

临时挂载：`sudo mount /dev/sda1 /mnt`

如需永久挂载，需编辑/etc/fstab文件


## 完整示例流程

假设你创建了 /dev/sda3 分区：

### 1. 重新读取分区表
`sudo partprobe /dev/sda`

### 2. 查看新分区
`sudo fdisk -l /dev/sda`

`lsblk`

### 3. 格式化为 ext4
`sudo mkfs.ext4 /dev/sda3`

### 4. 创建挂载点
`sudo mkdir /mnt/data`

### 5. 获取 UUID
`sudo blkid /dev/sda3`

假设 UUID 是 12345678-1234-1234-1234-123456789012

### 6. 编辑 fstab
`echo "UUID=12345678-1234-1234-1234-123456789012 /mnt/data ext4 defaults 0 2" | sudo tee -a /etc/fstab`

### 7. 测试挂载
`sudo mount -a`

### 8. 验证
`df -h /mnt/data`

`lsblk /dev/sda3`
